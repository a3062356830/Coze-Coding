#  Coze 术语锁定翻译系统技术方案（中 → 英）

##  111 项目目标

构建一个基于 **Coze 工作流** 的翻译系统，实现：

- 输入：中文全文
- 输出：英文全文
- 命中术语表中的词/短语必须使用固定译法
- 非术语内容由 LLM 自行翻译
- 目前库存**9999**条术语
- 命中率 ≥ 95%
- 数据库结构不需要其余校验，只考虑两列：中文、英文

---

## 🏗️ 一、系统架构
输入中文全文
↓
文本预处理
↓
术语扫描（插件）
↓
占位符替换
↓
文本切块
↓
LLM翻译（保护占位符）
↓
合并译文
↓
占位符回填（插件）
↓
术语完整性校验（插件）
↓
输出最终英文

---

## 🗄️ 二、数据结构

### 术语表：term_dict

| 中文       | 英文            |
|------------|-----------------|
| 扣子       | Coze            |
| 字节跳动   | ByteDance       |
| 飞书       | Lark            |
| 火山引擎   | Volcano Engine  |
| 豆包       | Doubao          |

> 所有复杂逻辑（归一化 / 冲突消解 / 匹配策略）在插件层实现

---

## ⚙️ 三、核心技术方案

### 3.1 匹配方式

- Trie
- Aho-Corasick

目标：对全文进行一次扫描，找出所有术语命中

---

### 3.2 匹配原则

1. 最长匹配优先  
2. 不允许重叠命中  
3. 同位置只保留一个术语  

---

### 3.3 占位符机制

[[T001]]
[[T002]]


要求：

- 不可被模型改写
- 唯一编号
- 可回填

---

## 🔍 四、术语识别与替换

### 输入

欢迎使用扣子，这是一款由字节跳动推出的AI工作平台。


### 输出

```json
{
  "protected_text": "欢迎使用[[T001]]，这是一款由[[T002]]推出的AI工作平台。",
  "term_map": {
    "[[T001]]": "Coze",
    "[[T002]]": "ByteDance"
  }
}
```


### 五、文本处理策略
##  5.1 预处理

全角转半角

去多余空格

括号统一

标点标准化

## 5.2 分段（segment）

流程顺序：

先术语识别

再文本切块

切块规则：

按段落切

长段按句号切

每块 300–800 字

不拆占位符

### 六、LLM 翻译策略

你是专业翻译助手。
请将输入的中文翻译为英文。

规则：
1. [[Txxx]] 占位符必须原样保留
2. 不得删除或改写占位符
3. 仅翻译自然语言内容
4. 保持结构与语义一致

输入：
{{chunk_text}}

```python
欢迎使用[[T001]]，这是[[T002]]推出的平台。
```

输出：

```python
Welcome to [[T001]], a platform launched by [[T002]].
```
